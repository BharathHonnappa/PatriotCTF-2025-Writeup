# Matrix Reconstruction - CTF Writeup

## Challenge Overview

This cryptography challenge involves recovering a matrix-based PRNG (Pseudorandom Number Generator) and using it to decrypt a ciphertext. The system uses a linear recurrence relation over GF(2) (binary field).

**Given:**
- Ciphertext (35 bytes)
- 40 leaked keystream states
- PRNG model: `S[n+1] = A*S[n] XOR B` over GF(2)

**Goal:** Recover matrix A and vector B, then decrypt the ciphertext.

## Understanding the Cryptosystem

### The PRNG Model

The keystream generator uses a linear feedback system:

- Operations are in GF(2) (XOR arithmetic)

## Solution Approach

### Step 1: Matrix Reconstruction

With 40 consecutive states, we can reconstruct matrix A and vector B by solving a system of linear equations over GF(2).

**Key Insight:** Each bit position in the output can be treated as an independent linear equation.

For each output bit position `i` (0-31):

This gives us 32 unknowns (the i-th row of A plus B[i]) per equation. With 40 states, we have 39 transitions, providing 39 equations per bit position - more than enough to solve for the 33 unknowns.

## Key Takeaways

1. **Linear Recurrences are Vulnerable**: When consecutive states leak, linear recurrence relations over GF(2) can be completely recovered using linear algebra.

2. **GF(2) Arithmetic**: Binary field operations (XOR-based) make the system linear and solvable with Gaussian elimination.

3. **Sufficient Data**: With 40 states and only 33 unknowns per row, we had redundant equations, making the system overdetermined and more robust.

4. **State Extension**: Once the PRNG is recovered, we can generate unlimited keystream, completely breaking the cipher.
